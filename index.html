Demonstrativo de Aferição de Remanejamento Orçamentário — Gestão de Parcerias OSC

1. Instruções de Uso

📋 Como preencher o Demonstrativo de Aferição

🗂 O que é este sistema?
O sistema de Aferição de Remanejamento Orçamentário serve para controlar e documentar as alterações entre os valores previstos e os efetivamente executados em uma parceria entre OSC (Organização da Sociedade Civil) e Unidade Gestora pública. Ele calcula automaticamente o PA (Percentual Acumulado) de alterações e indica se a execução está dentro do limite de autonomia de 15% sobre o Valor Total da Parceria (VTP).
Passo 1 — Cabeçalho (aba 2):
Preencha todos os campos antes de iniciar o lançamento de despesas:
  • Nome da OSC: razão social da entidade parceira.
  • Nome do Projeto: título completo conforme o instrumento jurídico.
  • Número do Termo: número do convênio, termo de fomento ou colaboração (máx. 30 caracteres).
  • Processo SEI: número do processo eletrônico no sistema SEI (máx. 20 caracteres).
  • VTP (Valor Total Previsto): valor global da parceria, sem incluir rendimentos. Este valor é o denominador constante de todos os cálculos de percentual. Não deve ser alterado durante a execução, salvo Termo Aditivo de valor.
  • Valor Repassado: montante efetivamente transferido para a conta da parceria.
  • Rendimentos: total de juros/aplicação acumulados na conta. O uso de rendimentos não conta no limite de 15%.
  • Data de Início / Fim: definem o número de meses do projeto e os cabeçalhos das colunas mensais nas tabelas.
Passo 2 — Previsto (aba 4):
Registre o plano orçamentário original aprovado no instrumento:
  • Clique em "+ Adicionar Item" para cada linha de despesa.
  • Selecione o Elemento de Despesa (Pessoal, Materiais, Administrativas, Serviços de Terceiros, Outras Despesas, Imobilizado, Implantação).
  • Informe a Quantidade (ex: nº de profissionais, unidades) — campo informativo.
  • Preencha o Item de Despesa com a descrição específica (ex: "Coordenador de projeto").
  • Lance os valores mês a mês. A coluna Total é calculada automaticamente.
  • Use ▲ / ▼ para reordenar linhas, "Inserir Acima" para incluir uma linha entre outras existentes.
  • Para apagar, selecione a linha e clique em "✕ Remover".
  • ⚠ O total previsto de todos os itens não deve ultrapassar o VTP.
Passo 3 — Executado (aba 5):
Registre os valores efetivamente gastos mês a mês:
  • As linhas do Previsto são espelhadas automaticamente — os campos de Elemento, Qtd e Item não são editáveis aqui.
  • Lance apenas os valores de execução em cada célula de mês.
  • Célula amarela: o valor executado excede o previsto naquele mês e item — isso gera PRA na aba de Alteração.
  • Para despesas não previstas no plano original, use "+ Só Executado". Essas linhas aparecerão em verde e também entram no cálculo do PA.
  • Remoção de linhas não é permitida nesta aba — para excluir um item, use a aba Previsto.
  • ⚠ O total executado não deve ultrapassar o Valor Repassado + Rendimentos.
Passo 4 — % de Alteração (aba 6):
Esta aba é somente leitura e calculada automaticamente:
  • PRA (célula) = max(0, (Executado − Previsto) / VTP). Células com PRA > 0 ficam destacadas em amarelo claro.
  • PA = soma de todas as células PRA. Representa o impacto percentual acumulado de remanejamento.
  • Se PA ≤ 15%: remanejamento dentro da autonomia → verde. Registre a justificativa no Demonstrativo de Aferição.
  • Se PA > 15%: limite excedido → vermelho. Requer consulta e anuência prévia da Unidade Gestora e possível Termo Aditivo.
⚠ Restrições de Integridade (Hard Blocks):
  • Vedação de Exclusão: é proibido zerar um item de despesa (executado = R$ 0,00) quando havia valor previsto, sem anuência expressa da Unidade Gestora.
  • Alteração de Cronograma: movimentação de valores entre parcelas mensais de um mesmo item exige Termo Aditivo, independentemente do percentual.
  • Rendimentos: só podem ser utilizados após o esgotamento integral dos recursos repassados.
💾 Salvar, Abrir e Exportar:
  • SALVAR (Ctrl+S): salva o projeto em formato .daf.json. Guarde este arquivo — ele contém todos os dados do projeto.
  • Abrir (Ctrl+O): carrega um arquivo .daf.json salvo anteriormente.
  • Salvar Como: salva com um nome diferente (útil para criar versões).
  • Exportar PDF: use o botão na barra superior para imprimir todas as abas, ou os botões "🖨 Imprimir esta seção" nas abas individuais para exportar apenas aquela tabela (recomendado para projetos com muitos meses).
  • Exportar CSV: gera 3 arquivos (Previsto, Executado, % Alteração) compatíveis com Excel.
  • Auto-save: o sistema salva um rascunho automaticamente no navegador. Se fechar sem salvar, será oferecida a restauração na próxima abertura.
📋 Colar do Excel (Ctrl+V):
  • Selecione as células no Excel e copie (Ctrl+C).
  • Clique na célula de destino na tabela do sistema.
  • Cole (Ctrl+V) — os valores serão distribuídos pelas linhas e colunas a partir da célula selecionada.
  • Aceita separador decimal vírgula (1.200,50) e ponto (1200.50).
  • Colunas de texto (Elemento, Item de Despesa) também podem ser coladas diretamente.
